# v0.3 Work Plan — Resources and Scale

Prepared 2026-07-07, after v0.2 landed. Scope from the roadmap: resources
compiled to data tables (sounds, curves, shapes), array state, and
multiple change-flag bytes. Target: the sprite-mover with an eased
slide-into-place, and the growth path opening to trail mode and snake.

## What v0.3 delivers

1. **`sound` resources. ✅ Landed 2026-07-08.** `sound Beep len 24 div 3` generates a
   callable `Snd_Beep` routine that loads the constants and jumps to
   `SndStart`. It is matrix-profile-only and intentionally limited to
   non-blocking low-frequency beeps/clicks.
2. **`curve` resources (P9). ✅ Landed 2026-07-08.** `curve SlideIn ease_out steps 64 from 0
to 7`: the compiler does the floating-point easing at build time and
   emits a page-aligned ROM table in destination space. A `compute`
   block maps a ramp through it with one indexed load. Presets: linear,
   ease_in, ease_out, ease_in_out, sine, overshoot, anticipation; an
   explicit value-list form for hand-tuned motion.
3. **`shape` resources. ✅ Landed 2026-07-08.** Pixel-art rows with a colour
   (`shape Dot color green` + `"..XX...."` rows), emitted as row-bitmap
   tables, drawn by a new library routine `ShapeDraw` (HL = shape
   pointer, B = x, C = y). Current shapes are matrix-profile-only,
   rectangular, 1..8 by 1..8, and unclipped. Multi-rotation variants wait
   for the Tetro milestone.
4. **Array state (P2). ✅ Landed 2026-07-08.** `state Board : byte[8]`:
   storage is `.ds N, 0`, one change flag for the whole array, legal in `on`
   and `updates`. Indexing is ordinary Z80 in blocks. `trail.glim` uses
   this as an 8-row mask buffer; snake and the Tetro board are now
   structurally possible.
5. **Multiple change-flag bytes. ✅ Landed 2026-07-08.** Cells beyond 8 allocate into
   `Changed1`/`Raised1`/`Next1` (and so on, capped at 4 banks = 32
   cells). Each block's trigger mask lives in one bank where possible;
   dispatch tests only the banks a block's triggers occupy. Rollover
   machinery repeats per bank.

## Design decisions to settle first

- **Colour names** in shape declarations: green/red/blue/yellow/cyan/
  magenta/white mapping to the COLOR_* composites. ✅ Settled for the
  matrix shape implementation.
- **Curve preset shapes**: quadratic ease in/out as the baseline;
  overshoot/anticipation amplitude (suggest 15% beyond the from/to
  range, clamped to byte bounds).
- **Bank assignment** for flags: declaration order fills bank 0 first
  (states, pulses, ramps, FrameCount), overflow to bank 1. ✅ Settled;
  dispatch tests only the banks a block actually depends on.
- **LCD text resources**: deferred to v0.4 unless the taste-tests want
  status text sooner — the LCD primitive library (busy/cmd/string) is a
  self-contained lift from the corpus when called for.

## Implementation order

Each step lands with tests and keeps `--rc strict --reg-profile mon3`
green.

1. ✅ `sound` resource + `Snd_` reserved prefix + slide.glim's beep
   becomes a declaration.
2. ✅ `curve` resource: build-time table generation, `.align 256`, the
   `from/to` destination-space scaling; `slide.glim` now uses an
   ease-out curve for arrival.
3. ✅ `shape` resource + `ShapeDraw` library routine; the slide dot is
   now a generated 2x2 shape.
4. ✅ Array state; `trail.glim` — the dot leaves a trail in a
   `byte[8]` row-mask array.
5. ✅ Multi-bank change flags; tests cover cells spilling into bank 1 and
   the 32-cell cap.
6. ✅ Docs: grammar implemented section, manual chapter, engineering
   manual, roadmap, and this plan all reflect landed v0.3.

## Acceptance

- ✅ slide.glim uses a build-time ease-out curve for the arrival path.
- ✅ trail.glim assembles strict-clean; snake becomes writable with nothing but v0.3
  features (snake itself is the v0.4-era demo).
- ✅ All tracked tests pass; coverage is above the v0.3 target of 25 tests.

## Carried risks and notes

- Word-state value-compare (flag only when the value actually changed)
  stays deferred: unconditional flagging is correct, compare is an
  optimization.
- `ShapeDraw`'s per-row shift is the first library routine with a real
  inner loop; worth watching frame budget on hardware.
- Curve tables at 256-byte alignment cost padding; acceptable at these
  program sizes, revisit if ROM pressure appears.
